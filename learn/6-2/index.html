⬡ STATION 01 — THEORY + PRACTICE

Data Formats

Master CSV, JSON, XML and Fixed-Width — and when to use each.

CSV
📄
Comma-Separated Values
Plain-text rows and columns separated by commas. Lightweight, human-readable, no overhead. Best for: flat tabular data, bulk export, spreadsheet import. Cannot represent nested or hierarchical data.
EXAMPLEid,name,spend / 001,Alice,420
JSON
🔗
JavaScript Object Notation
Lightweight key-value pairs with native support for nested arrays and objects. Best for: web APIs, hierarchical data, mobile apps. Lighter than XML. Strict syntax — one missed comma breaks the file.
EXAMPLE{"name":"Alice","orders":[{"item":"Phone"}]}
XML
📋
Extensible Markup Language
Self-describing tags wrapping every value. Verbose — opening AND closing tags make files significantly larger than JSON. Best for: legacy enterprise integration, SOAP services, configuration files. Key weakness: verbosity increases bandwidth costs.
KEY TERMVerbose — opening & closing tags increase file size
FIXED-WIDTH
📏
Predetermined Field Lengths
Each field always occupies a fixed number of characters — shorter values are padded with spaces. Fast parsing (no delimiter search). Best for: legacy banking, mainframe batch processing, government data transfers. Wastes storage due to padding.
EXAMPLE001 Alice a@co.uk 420.00

TERM
CSV
DEFINITION
A plain-text format storing tabular data as comma-separated rows and columns. Lightweight, human-readable, ideal for flat non-hierarchical data.
TERM
JSON
DEFINITION
A lightweight key-value format with native support for nested arrays and objects. The standard format for web APIs. Lighter than XML.
TERM
XML Verbosity
DEFINITION
XML uses opening AND closing tags around every value, significantly increasing file size compared to JSON. This increases bandwidth usage and transfer time at scale.
TERM
Fixed-Width Format
DEFINITION
A format where every field occupies a predetermined number of characters, with shorter values padded by spaces. Used in legacy banking and mainframe systems.
COMPARE
JSON vs XML — key difference
ANSWER
Both support hierarchical data. JSON is lighter (no closing tags), human-readable and native to web APIs. XML is verbose but self-describing and widely supported in legacy enterprise systems.
COMPARE
CSV vs Fixed-Width — key difference
ANSWER
Both store flat data. CSV uses commas as delimiters — compact but commas in values cause errors. Fixed-width uses character positions — predictable for legacy parsers but wastes space on padding.

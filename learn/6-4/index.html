⬡ STATION 01 — THEORY + PRACTICE

Data Security and Management

Master the CIA triad, GDPR, encryption and access controls.

THE CIA TRIAD
🛡️
Three Security Goals
Confidentiality — only authorised users can access data. Integrity — data is accurate and has not been tampered with. Availability — data is accessible to authorised users when needed. Every security measure should be evaluated against which CIA goal it protects.
EXAM TIPRansomware primarily attacks Availability — not Confidentiality
GDPR PRINCIPLES
📜
Name the Specific Principle
Key principles: Lawfulness/fairness/transparency — tell people what you collect. Purpose limitation — only use data for its stated purpose. Data minimisation — collect only what is necessary. Storage limitation — do not keep data longer than needed. Accuracy — keep data up to date.
EXAMINER TIP"Follow GDPR" scores zero — name the specific principle with an example
ENCRYPTION
🔒
Converting Data to Ciphertext
Encryption transforms readable data into unreadable ciphertext — only the holder of the correct key can decrypt it. Symmetric: same key encrypts and decrypts (fast, for stored data). Asymmetric: public key encrypts, private key decrypts (secure transmission, HTTPS/TLS). Always name the specific threat encryption prevents.
KEY TERMEncryption in transit (TLS) vs encryption at rest (database encryption)
ACCESS CONTROLS
🔑
Principle of Least Privilege
Role-Based Access Control (RBAC): access rights assigned by job role. Principle of least privilege: users should only have access to the minimum data required for their role. This limits the blast radius of any breach. Example: a sales rep can view customer contacts but cannot access HR salary data.
BREACH CONSEQUENCEICO must be notified within 72 hours — failure is itself a GDPR violation

TERM
Confidentiality (CIA)
DEFINITION
Only authorised users can access data. Protected by encryption, access controls and authentication.
TERM
Availability (CIA)
DEFINITION
Data is accessible to authorised users when needed. Threatened by ransomware, DDoS attacks and system failures.
TERM
Purpose Limitation (GDPR)
DEFINITION
Data must only be used for the specific purpose for which it was collected. Using email addresses for marketing when collected for order confirmations violates this principle.
TERM
Principle of Least Privilege
DEFINITION
Users should only have access to the minimum data and systems required for their specific role, limiting the damage any breach or insider threat can cause.
TERM
Encryption in Transit
DEFINITION
Protecting data as it moves across networks using protocols like TLS/HTTPS. Intercepted traffic appears as unreadable ciphertext without the decryption key.
TERM
Data Breach (GDPR)
DEFINITION
An incident where personal data is accessed, disclosed or lost without authorisation. Must be reported to the ICO within 72 hours if likely to risk individuals.
